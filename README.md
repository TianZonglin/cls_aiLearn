# 博士实践课：用 AI 做一个工具

本仓库用于博士实践课的课程资料整理与实践过程记录。

本次课程周期共四周，目标是每位同学都使用 AI 完成一个可用的小工具。课程会围绕工具安装、AI 工具使用、任务设计、开发实践和进度交流逐步展开。

## 课程目标

每位同学完成一个基于 AI 的工具，可以是学习辅助、数据处理、文档整理、自动化脚本、网页应用或其他能够解决实际问题的小工具。

最终成果建议包括：

1. 工具说明文档。
2. 可运行的代码或配置文件。
3. 使用截图或演示材料。
4. 简短的实践总结。

## 时间安排

### 第 1 周：工具准备

本周先不上课。

老师会发布软件清单，同学们先根据清单安装需要用到的工具。没有上过 AI 博士课的同学需要完整安装；之前上课时没有安装好的同学也需要补充安装。

安装教程见：

[softwares_install.md](./softwares_install.md)

### 第 2 周前：安装问题处理

如果安装过程中遇到问题，可以在下周正式上课前联系老师处理。

支持方式包括：

1. 线上沟通。
2. 到新创 1402 办公室现场处理。

### 第 2 周至第 4 周：任务布置与实践交流

从下周五开始，计划用 2-3 次课完成以下内容：

1. 布置实践任务。
2. 学习和熟悉 AI 工具。
3. 交流选题与开发进度。
4. 解决实践过程中的问题。
5. 推进每位同学完成自己的 AI 工具。

## 准备事项

请同学们在正式上课前完成以下准备：

1. 按照软件清单安装所需工具。
2. 验证命令行、Node.js、Git、GitHub 和 Codex 等工具是否可用。

4. 初步思考自己想用 AI 做什么工具。
5. 记录安装或使用中遇到的问题，方便课前集中处理。

## 选题建议

工具选题不要求复杂，重点是能用 AI 解决一个明确的小问题。例如：

1. 自动整理课程资料。
2. 批量生成或修改文档。
3. 自动分析表格数据。
4. 生成学习计划或复习提纲。
5. 做一个简单的网页工具。
6. 做一个自动化脚本，减少重复操作。

建议优先选择自己真实需要、范围清晰、四周内能完成的题目。
