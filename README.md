# 博士实践课：用 AI 做一个工具

本仓库用于博士实践课的课程资料整理与实践过程记录。

本次课程周期共四周，目标是每位同学都使用 AI 完成一个可用的小工具。课程会围绕工具安装、AI 工具使用、任务设计、开发实践和进度交流逐步展开。

## 课程目标

每位同学完成一个基于 AI 的工具，可以是学习辅助、数据处理、文档整理、自动化脚本、网页应用或其他能够解决实际问题的小工具。

最终成果建议包括：

1. 工具说明文档。
2. 可运行的代码或配置文件。
3. 使用截图或演示材料。
4. 简短的实践总结。

## 选题建议工具

选题不要求做成大型系统，但建议比单次问答更完整一些，最好包含“输入、处理、输出、保存或展示”的完整流程。例如：

1. 课程资料自动整理工具：批量读取课件、文档和课堂笔记，自动提取主题、关键词和摘要，并按课程、日期或知识点生成目录。
2. 文献阅读辅助工具：上传论文 PDF 后，自动生成研究问题、方法、数据、结论、创新点和不足，并支持导出 Markdown 阅读笔记。
3. 表格数据分析助手：导入 Excel 或 CSV 文件后，自动识别字段含义，生成基础统计、异常值提示、图表建议和一段自然语言分析报告。
4. 访谈或会议纪要生成工具：输入录音转写文本后，自动整理议题、关键观点、待办事项、负责人和截止时间。
5. 个人知识库问答工具：将课程资料、论文笔记和项目文档整理成知识库，支持用自然语言提问并返回带来源的答案。
6. 实验记录与报告生成工具：根据实验数据、过程描述和图片，自动生成实验记录、结果分析和报告初稿。
7. 简历或项目材料优化工具：输入个人经历和目标岗位，自动生成简历条目、项目描述、面试问答和修改建议。
8. 小型网页应用生成器：输入功能描述后，自动生成一个可运行的网页工具，例如计算器、资料检索页、问卷分析页或学习计划生成器。
9. GitHub 项目说明生成工具：读取代码目录和主要文件，自动生成 README、安装说明、使用示例和常见问题。
10. 作业批改辅助工具：导入学生提交内容后，按照评分标准生成评语、扣分点、改进建议和成绩汇总表。

建议优先选择自己真实需要、范围清晰、四周内能完成的题目。
