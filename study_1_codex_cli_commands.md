# Codex CLI 斜杠命令表格速查

本文面向博士实践课同学，把 Codex CLI 交互界面中常用的斜杠命令整理成表格，方便上课、练习和排查问题时快速查阅。

参考文档：

- <https://www.codex-docs.com/using-codex/cli/slash-commands/>

说明：Codex CLI 更新较快，不同版本能看到的命令可能略有差异。如果实际界面和本文不完全一致，以你在 Codex 输入框中输入 `/` 后弹出的命令列表为准。

# 1. 新手最该先记住的工作流

| 顺序 | 命令 | 作用 | 什么时候用 | 示例 |
|---|---|---|---|---|
| 1 | `/plan` | 先规划，不急着改文件 | 任务复杂、需求不清楚、想先看方案 | `/plan 请先设计一个课程资料整理工具方案` |
| 2 | `/permissions` | 控制 Codex 的读写和执行权限 | 修改文件前、运行命令前 | `/permissions` |
| 3 | `/status` | 确认模型、目录、权限、上下文 | 动手前、切模型后、长对话中 | `/status` |
| 4 | `/model` | 根据任务难度切换模型 | 任务变复杂或想加快响应 | `/model` |
| 5 | `/diff` | 查看 Codex 改了哪些文件 | 修改完成后、提交前 | `/diff` |
| 6 | `/review` | 提交前审查当前改动 | 改动较大、准备提交时 | `/review` |
| 7 | `/compact` | 压缩长对话上下文 | 对话很长但任务还没结束 | `/compact` |
| 8 | `/quit` | 退出 Codex | 任务完成后 | `/quit` |

# 2. 斜杠命令基础概念

| 项目 | 说明 | 示例 | 注意事项 |
|---|---|---|---|
| 什么是斜杠命令 | Codex CLI 交互界面中的快捷控制命令 | `/status`、`/diff`、`/review` | 必须进入 Codex 后输入 |
| 如何触发 | 在 Codex 输入框中输入 `/` | `/` | 会弹出当前版本支持的命令列表 |
| 主要作用 | 控制当前 Codex 会话 | 切模型、查状态、看改动、审查代码 | 不是普通系统命令 |
| 学习方式 | 先掌握高频命令 | `/plan`、`/status`、`/diff` | 不需要一次背完所有命令 |

# 3. 斜杠命令和终端命令的区别

| 类型 | 输入位置 | 作用 | 示例 | 适合场景 |
|---|---|---|---|---|
| 终端命令 | `cmd`、PowerShell、Git Bash、macOS/Linux 终端 | 启动 Codex、查看版本、执行一次性任务 | `codex`、`codex --version`、`codex review` | 进入 Codex 前使用 |
| 斜杠命令 | Codex 已启动后的输入框 | 控制当前 Codex 会话 | `/status`、`/diff`、`/review` | 进入 Codex 后使用 |
| 启动指定模型 | 终端 | 启动前指定模型 | `codex -m 模型名` | 新开会话前就确定模型 |
| 会话中切模型 | Codex 输入框 | 不重启，直接切换当前模型 | `/model` | 任务中途变复杂或需要调整速度 |

# 4. 如何打开和使用斜杠命令

| 操作 | 输入 | 作用 | 适合场景 |
|---|---|---|---|
| 打开命令菜单 | `/` | 弹出命令列表 | 不知道当前支持哪些命令 |
| 搜索命令 | `/sta` | 筛选类似 `/status` 的命令 | 只记得命令前几个字母 |
| 执行命令 | `/status` 后回车 | 运行该斜杠命令 | 已确认要执行的命令 |
| 任务运行时排队 | 输入命令后按 `Tab` | 把命令排到下一轮执行 | Codex 正在运行，想之后立刻 `/diff` 或 `/review` |

# 5. 核心命令速查

| 命令 | 作用 | 什么时候用 | 常见搭配 |
|---|---|---|---|
| `/plan` | 先规划方案 | 复杂任务、需求不清楚 | `/plan` 后确认方案再执行 |

| `/compact` | 压缩上下文 | 对话很长但任务没完 | `/status` |

# 6. `/plan` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/plan` |
| 作用 | 让 Codex 先给方案，不立刻修改文件 |
| 适合场景 | 需求不清楚、任务涉及多个文件、担心直接改偏、需要先给老师或同学看方案 |
| 基本用法 | `/plan` |
| 带任务用法 | `/plan 请帮我设计一个课程资料自动整理工具，要求四周内能完成。` |
| 推荐后续 | 确认方案后再说：`方案可以，先实现第一版最小可运行功能。` |
| 注意事项 | `/plan` 适合“先想清楚”，不是最终交付命令 |

# 7. `/permissions` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/permissions` |
| 作用 | 调整 Codex 的读写、执行和审批方式 |
| 适合场景 | 第一次打开项目、准备修改文件、准备运行命令、正式项目中需要控制风险 |
| 基本用法 | `/permissions` |
| 保守模式 | 让 Codex 先阅读和分析，不直接改文件 |
| 需要确认 | 关键操作前让你批准 |
| 更自动 | 允许 Codex 主动修改和执行，适合练习项目或小范围任务 |
| 推荐提示 | `请阅读当前项目，说明目录结构和主要文件作用，不要修改文件。` |

# 8. `/status` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/status` |
| 作用 | 查看当前会话状态 |
| 重点查看 | 当前模型、工作目录、可写目录、权限模式、上下文使用情况、Codex 版本 |
| 基本用法 | `/status` |
| 修改文件前 | 先确认工作目录是否正确 |
| 切换模型后 | 用 `/status` 确认模型是否切换成功 |
| 长对话中 | 查看上下文压力，必要时执行 `/compact` |
| 目录不对时 | 退出 Codex，进入正确目录后重新运行 `codex` |

---

# 9. `/model` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/model` |
| 作用 | 在当前会话中切换模型 |
| 适合场景 | 任务变复杂、任务很简单、需要二次审查、想调整响应速度 |
| 基本用法 | `/model` |
| 任务复杂时 | 切换到更强模型 |
| 任务简单时 | 切换到更快或更轻量的模型 |
| 二次审查时 | 让另一个模型检查方案、风险和遗漏 |
| 和 `codex -m` 区别 | `codex -m` 是启动前指定模型；`/model` 是会话中途切换 |

---

# 10. `/diff` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/diff` |
| 作用 | 查看当前工作区的文件改动 |
| 适合场景 | Codex 修改文件后、提交 Git 前、担心改了不该改的地方 |
| 基本用法 | `/diff` |
| 推荐追问 | `请解释这次 diff：每个文件改了什么、为什么改、有没有风险。` |
| 和 Git 的关系 | `/diff` 适合在 Codex 内继续讨论；`git diff` 是标准 Git 工作流 |
| 常见搭配 | `/diff` 后执行 `/review` |

---

# 11. `/review` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/review` |
| 作用 | 审查当前工作树中的改动，重点找问题、风险和遗漏 |
| 适合场景 | 提交前、多文件修改后、修复 bug 后、想确认有没有漏测 |
| 基本用法 | `/review` |
| 推荐流程 | `/status` → `/diff` → `/review` |
| 处理结果 | 不建议一次全改，优先逐个处理 review 中的重要问题 |
| 推荐追问 | `请先修复 review 中第 1 个问题，其他问题暂时不要动。` |

# 12. `/mention` 速查

| 项目 | 内容 |
|---|---|
| 命令 | `/mention` |
| 作用 | 把某个文件或目录明确加入当前对话，让 Codex 重点关注 |
| 适合场景 | 只想改某个文件、项目文件很多、让 Codex 解释某个配置文件、根据某份文档继续写 |
| 指定文件 | `/mention README.md` |
| 指定目录 | `/mention docs` |
| 推荐提示 | `/mention study_1_codex_cli_commands.md` 后输入：`请检查这份教程是否适合初学者阅读。` |
| 注意事项 | 文件名越明确，Codex 越不容易找错目标 |

# 13. 上下文管理命令速查

| 命令 | 作用 | 什么时候用 | 示例 | 注意事项 |
|---|---|---|---|---|
| `/compact` | 压缩前面的长对话，释放上下文空间 | 当前任务没完成，但对话已经很长 | `/compact` | 会尽量保留关键信息 |
| `/new` | 在同一个 CLI 中开启新对话 | 当前任务完成，要换新任务 | `/new` | 旧上下文不再影响新任务 |
| `/clear` | 清空终端显示并开启新聊天 | 界面太乱，同时想开始新任务 | `/clear` | `Ctrl + L` 只是清屏，不等于 `/clear` |

# 14. 会话继续和分支命令速查

| 命令 | 作用 | 什么时候用 | 示例 | 注意事项 |
|---|---|---|---|---|
| `/resume` | 恢复历史会话 | 继续上次没做完的任务 | `/resume` | 适合跨天继续工作 |

# 16. 其他常用斜杠命令速查

| 命令 | 作用 | 什么时候用 | 示例 | 注意事项 |
|---|---|---|---|---|
| `/fast` | 切换快速模式 | 想提高响应速度时 | `/fast on`、`/fast off`、`/fast status` | 不一定所有模型都支持 |
| `/personality` | 调整回复风格 | 想让 Codex 更简洁、更友好或更务实时 | `/personality` | 影响后续回复风格 |
| `/goal` | 设置较长任务目标 | 希望 Codex 持续围绕一个目标推进 | `/goal Finish the tutorial rewrite` | 适合长任务，不适合一句话小任务 |
| `/mcp` | 查看可用 MCP 工具 | 需要确认外部工具是否可用 | `/mcp`、`/mcp verbose` | 入门阶段不必重点掌握 |
| `/init` | 生成 `AGENTS.md` | 给项目添加协作规则、测试命令和说明 | `/init` | 适合正式项目 |

| `/keymap` | 调整键位方案 | 想修改输入快捷键时 | `/keymap` | 适合熟悉 CLI 后再用 |
| `/theme` | 调整界面主题 | 想切换显示风格时 | `/theme` | 只影响界面显示 |
| `/debug-config` | 排查配置、权限策略、MCP 加载等问题 | Codex 配置异常时 | `/debug-config` | 普通使用中较少需要 |
| `/quit` | 退出 Codex CLI | 任务完成后 | `/quit` | 和 `/exit` 类似 |
| `/exit` | 退出 Codex CLI | 任务完成后 | `/exit` | 和 `/quit` 类似 |

# 17. 常见使用场景速查

| 场景 | 推荐命令 | 后续可以输入的话 | 目标 |
|---|---|---|---|
| 第一次打开项目 | `/permissions` → `/status` | `请阅读当前项目，说明目录结构和主要文件作用，不要修改文件。` | 先理解项目，不急着动手 |
| 写一个新教程 | `/plan` | `请帮我设计一份 Codex CLI 命令教程的大纲，面向初学者。` | 先定结构，再写正文 |
| 修改 README | `/mention README.md` | `请把选题建议部分改得更具体一些。` | 聚焦指定文件 |
| 提交前检查 | `/status` → `/diff` → `/review` | `请解释这次 diff，并指出提交前还要确认什么。` | 降低提交风险 |
| 对话很长但还要继续 | `/compact` | `继续完成剩余任务，并在最后说明改了哪些文件。` | 保留关键信息，继续推进 |
| Codex 改了文件但看不懂 | `/diff` | `请用初学者能看懂的方式解释这次 diff。` | 理解改动内容和风险 |
| 任务中途变复杂 | `/model` → `/status` | `请重新评估这个任务的实现步骤和风险。` | 换模型后确认状态 |

# 18. 常见问题速查

| 问题 | 推荐操作 | 示例 | 说明 |
|---|---|---|---|
| 不知道当前支持哪些命令 | 输入 `/` 查看列表 | `/` | 以弹出的命令列表为准 |
| `/plan` 暂时不能用 | 等当前任务结束后再用 | `/plan` | 任务运行中有些命令可能暂时不可用 |
| `/model` 里没有想要的模型 | 先查状态和版本 | `/status`、`codex --version` | 可能是账号权限、提供商、版本或配置限制 |
| Codex 工作目录不对 | 先查状态，再重新进入正确目录 | `/status` | 目录错了就退出后重新 `cd` 到正确目录 |
| Codex 改了文件但看不懂 | 看 diff 并让 Codex 解释 | `/diff` | 可以要求解释每个文件为什么改 |
| 提交前不知道做什么 | 至少执行 `/diff` 和 `/review` | `/diff`、`/review` | 大改动建议再加 `/status` |
| PowerShell 运行 `codex` 报脚本策略错误 | 改用 cmd 运行 | `cmd /c codex` | 也可以直接打开“命令提示符 cmd”后输入 `codex` |

# 一句话总结

| 学习阶段 | 优先掌握 |
|---|---|
| 刚开始 | `/plan`、`/permissions`、`/status` |
| 开始修改文件 | `/mention`、`/diff` |
| 提交前 | `/review` |
| 对话变长 | `/compact` |
| 任务完成 | `/quit` 或 `/exit` |

最推荐的新手顺序：

```text
/plan -> /permissions -> /status -> /model -> /diff -> /review -> /compact -> /quit
```
